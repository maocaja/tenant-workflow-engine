Motor de Workflow — C4

El mismo sistema en tres niveles de zoom. Cada nivel responde una pregunta distinta, para un lector distinto: el sistema completo en su mundo, las piezas ejecutables dentro de él, y las partes dentro de una pieza.

Nivel 1 Contexto del sistema

El sistema en su mundo

El mayor zoom: una caja para toda la plataforma, quién la usa y con qué habla. Un interesado no técnico debería entender esto. Aquí: un llamador de la API lleva los documentos por el flujo, y la plataforma emite eventos de aprobación que sistemas downstream (pagos, notificaciones) consumen.

Nivel 2 Contenedores

Las piezas ejecutables dentro de la plataforma

Zoom dentro de la caja. Un contenedor es algo que corre y guarda estado — un servicio, una base de datos, una cola — cada uno con su tecnología. Esta es la vista más útil para un ingeniero: muestra los dos servicios, sus bases de datos separadas y la cola con su cola de mensajes muertos. Cada servicio tiene su propia base — nada de esquema compartido.

Nivel 3 Componentes

Dentro de workflow-service

Zoom dentro de un contenedor. Los componentes son las agrupaciones grandes de código y cómo dependen entre sí — esta es la estructura hexagonal / puertos-y-adaptadores. Las dependencias apuntan hacia adentro: infraestructura → aplicación → dominio, y el dominio no depende de nada.

Nivel 4 Código

El repositorio es el cuarto nivel

C4 se detiene en el código porque el código es el código — se lee, no se diagrama. El componente Dominio es src/main/java/com/mauricio/workflow/domain/; los Casos de uso son application/; los adaptadores viven bajo infrastructure/. El consumidor es su propio módulo, consumer-service/.
